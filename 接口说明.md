# 华创形态学 Skill 接口说明

本文档说明 `huachuang-morphology` 中 **Python 客户端方法** 与 **底层 HTTP 接口** 的对应关系，以及 **CLI 调用方式** 与 **返回 JSON 结构**。实现代码见 `scripts/morphology_api.py`、`scripts/morphology_cli.py`。

---

## 1. 总览

| 项目 | 说明 |
|------|------|
| 认证 | 凡访问 `https://mark.hcquant.com/all_api/` 下脚本，均需在 Query 中传 `token`（来自 `scripts/config.json`）。 |
| 主域名 A | `https://mark.hcquant.com/all_api` — 基础数据、截面择时、ETF 相关 PHP 接口。 |
| 主域名 B | `https://xingtai.pro` — 历史择时 JSON、ETF 门户/得分等；多数为 **GET 固定路径**，**URL 中不带 token**（由服务端公开提供）。 |
| 请求方式 | 均为 **GET**（`requests.get`）。 |
| SSL | 客户端默认 `verify=False`（与历史环境兼容），生产环境请遵循贵司安全规范。 |

---

## 2. CLI 统一约定

在 **Skill 根目录**（与 `SKILL.md` 同级）执行：

```bash
python scripts/morphology_cli.py <方法名> '[JSON 参数字符串]'
```

| 规则 | 说明 |
|------|------|
| 方法名 | 与下表「Skill 方法名」一致，须为 `get_*` 或 `evaluate_etf_performance`。 |
| 第二参数 | 可选；为 **JSON 对象** 字符串，键名与 Python 参数名一致；未传的参数使用代码默认值。 |
| 列出方法 | `python scripts/morphology_cli.py --list` |
| 配置 | 必须存在 `scripts/config.json` 且含非空 `token`（`--list` 除外）。 |

### 2.1 标准输出 JSON 信封

成功或失败均在 **stdout** 打印一行 JSON（UTF-8，中文不转义为 `\uXXXX`）。

| 字段 | 含义 |
|------|------|
| `ok` | `true` 表示业务成功返回；`false` 表示失败或无数据。 |
| `error` | `ok=false` 时的人类可读说明。 |
| `type` | `ok=true` 时：`dataframe` \| `dict` \| `text` \| `json`。 |
| `rows` | 当 `type=dataframe` 时，表格行数。 |
| `data` | 载荷：`dataframe` 时为对象数组（每行一条记录）；`dict` 时为嵌套对象（内含 `processed_time` 等序列化后的数组）。 |

解析失败、网络错误等也可能在 **stderr** 输出 JSON 错误信息并以非 0 退出码结束。

## 3. 接口一览（按 Skill 方法）

下列「HTTP」列为逻辑路径；实际请求由 `base_url` + 路径或完整 `timing_base_url` URL 组成。

### 3.1 基础数据（需 token）

| Skill 方法名 | HTTP | Query 参数 | 默认 / 说明 | 返回类型（Python） |
|--------------|------|-------------|-------------|-------------------|
| `get_basic_info` | `GET .../basic.php` | `token` | — | `DataFrame` |
| `get_supported_assets` | `GET .../get_sid.php` | `token` | — | `DataFrame` |
| `get_latest_daily_stats` | `GET .../all.php` | `token` | — | `DataFrame` |
| `get_historical_data` | `GET .../fullfun.php` | `token`, `win`, `tr`, `pos` | `win_rate`→`win`（默认 `0.6`）；`trade_date`→`tr`（默认当天 `YYYY-MM-DD`）；`position`→`pos`（默认 `"负面"`，可选如 `"正面"`） | `DataFrame` |

### 3.2 截面择时（需 token）

| Skill 方法名 | HTTP | Query | 返回 |
|--------------|------|--------|------|
| `get_broad_index_timing` | `GET .../indextiming.php` | `token` | `DataFrame` |
| `get_industry_timing` | `GET .../indtiming.php` | `token` | `DataFrame` |
| `get_style_timing` | `GET .../styletiming.php` | `token` | `DataFrame` |
| `get_concept_timing` | `GET .../concepttiming.php` | `token` | `DataFrame` |

### 3.3 历史择时 JSON（xingtai，无 token）

请求 URL 模式：`GET https://xingtai.pro/<子路径>/<asset_code>.json`

客户端成功解析后，若存在 `betime`，会附加 **`processed_time`**：`DataFrame`，列为 `begindate`、`enddate`（已由 Unix 毫秒转为 UTC 时间）。

| Skill 方法名 | URL 模式 | 参数 | 默认 `asset_code` |
|--------------|----------|------|-------------------|
| `get_historical_broad_timing` | `/timing/{asset_code}.json` | `asset_code` | `881001.WI` |
| `get_historical_industry_timing` | `/indtiming/{asset_code}.json` | `asset_code` | `CI005001.WI` |
| `get_historical_style_timing` | `/styletiming/{asset_code}.json` | `asset_code` | `000015.SH` |
| `get_historical_concept_timing` | `/concepttiming/{asset_code}.json` | `asset_code` | `884030.WI` |

**响应体（典型）**：JSON 对象，常含 `code`、`msg`、`data`（K 线/序列数组）、`betime`（`[begin_ms, end_ms]` 区间列表）。具体字段以服务端为准。

### 3.4 ETF 相关

| Skill 方法名 | HTTP | 参数 | 说明 |
|--------------|------|------|------|
| `get_etf_cross_section_signal` | `GET .../etfnow.php` | `token`, `company` | `company` 默认 `fg`；公司代码见下文「附录」 |
| `get_etf_historical_signal` | `GET https://xingtai.pro/{folder}/{asset_code}.json` | `asset_code`, `company` | `folder` 由 `company` 映射：`th`→`etftimingth`，`htbr`→`etftiminghtbr`，`fg`/`bs`→`etftiming`，其余默认 `etftiming` |
| `get_etf_portal_data` | `GET .../etftimingV2/{sid}.json` | `sid` | 默认 `000037.SH`；`sid` 为跟踪指数代码 |
| `get_etf_scores` | `GET .../etfv2/score_all.json` | 无 | 返回 `DataFrame` |

### 3.5 绩效评价（依赖 empyrical）

| Skill 方法名 | 数据源 | 参数 | 返回 |
|--------------|--------|------|------|
| `evaluate_etf_performance` | `GET https://xingtai.pro/etftiming/{asset_code}.json` | `asset_code`（默认 `000812.CSI`） | **字典**：策略/基准的年化收益、最大回撤、夏普、Sortino、Calmar 及 `rows`（样本长度）。未安装 `empyrical` 时返回 `None`。 |

说明：该方法固定走 **`etftiming`** 路径，与 `get_etf_historical_signal` 中按 `company` 切换文件夹的逻辑**不同**；适用于与 `etftiming` 数据源一致的标的。

### 3.6 其他（需 token 或混合）

| Skill 方法名 | HTTP | Query / 路径 | 参数 |
|--------------|------|----------------|------|
| `get_realtime_historical_data` | `GET .../fullfun_realtime.php` | `token`, `tr` | `trade_date`→`tr`，默认当天 |
| `get_index_median_data` | `GET .../index_median_data.php` 或 `HK_median_data.php` | `token`, `sid` | `sid` 含子串 `HI` 时走港股接口 |
| `get_etf_median_data` | `GET .../etf_median_data_all.php` | `token`, `company`, `sid` | 默认 `company=bs`，`sid=000861.CSI` |

## 4. CLI 调用示例

```bash
# 基础
python scripts/morphology_cli.py get_basic_info

# 历史全市场（胜率、日期、持仓方向）
python scripts/morphology_cli.py get_historical_data '{"win_rate":0.6,"trade_date":"2024-01-02","position":"负面"}'

# 沪深300 历史宽基择时 JSON
python scripts/morphology_cli.py get_historical_broad_timing '{"asset_code":"000300.SH"}'

# ETF 截面（富国）
python scripts/morphology_cli.py get_etf_cross_section_signal '{"company":"fg"}'

# 指数中间数据
python scripts/morphology_cli.py get_index_median_data '{"sid":"000300.SH"}'
```

Windows PowerShell 中若外层用双引号，内层 JSON 双引号需转义为 `\"`，或使用单引号包裹整个 JSON 参数。
